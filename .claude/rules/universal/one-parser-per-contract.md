# One canonical parser per shared contract — and sweep every syntactic form when normalizing literals

When the same wire/config contract (env-var JSON keys, event fields, file formats) is parsed at more than one site, hoist the field names into a single canonical in-tree definition (typed record + parse method) and route every consumer through it. When the contract's true owner lives in an unreferencable project (a sidecar, a standalone tool), the canonical type's doc comment names that owner as the keep-in-sync source of truth.

**Why:** `STEAM_ACCOUNTS` entries were hand-parsed at four test-side sites, and the field names had drifted at three of them: `CollectKnownSecrets` and `SteamAccountSlicer`'s shape hint both said `refreshToken` — a key from the steam-service sidecar's *session.json* format, never valid in `STEAM_ACCOUNTS` (the sidecar's `SteamAccountConfig` binds `user`/`pass`/`token` case-sensitively and silently ignores unknown keys). The drift was runtime-invisible: configured refresh tokens simply never got masked by `ScrubForLog`/`ReportRedactor`. Fixed by the canonical `SteamAccount` record (`tests/JunimoServer.Tests/Schema/Json/SteamAccount.cs`), whose doc names the sidecar as the contract owner. Same session, same disease smaller: CIRenderer's failure-display caps existed as `Take(10)` at two sites *and* as an `i < 5` loop bound — a `Take(` grep alone missed one of the four cap sites.

**How to apply:** Before hand-parsing a shared contract (a `JsonDocument`/`JsonNode` walk, key-string lookups), grep for an existing canonical type and use it; if none exists and you are writing the second parser, create the type and rewire the first. Silently-ignored unknown keys (System.Text.Json's default binding) make this drift class invisible at runtime — don't expect a failing test to catch it. A pass-through site that must re-serialize entries verbatim may stay node-based, but its key names and shape hints still come from the canonical type. And when hoisting any duplicated literal into a named constant, enumerate its syntactic forms (`.Take(N)`, loop bounds `i < N`, interpolated strings) and grep each — not just the form you first saw. Sibling of `one-writer-per-artifact.md` (two producers of one artifact); this rule is the consumer/parser side of the same drift disease.
