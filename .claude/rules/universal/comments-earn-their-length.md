# Comments earn their length — why, not what; cut anything the code or name already says

A comment is justified only by what the reader can't get from the code itself: the *why*, the non-obvious constraint, the gotcha. Cut everything else. Specifically: don't restate what the next line does, don't repeat information the symbol/step/file name already carries, don't give precise figures where a rough magnitude conveys the point, and don't explain the same rationale in two places. When a comment runs several times the length of the code it annotates, that's the signal to compact it.

**Why:** Across one session the user compacted my comments repeatedly, each time on a distinct axis: a 12-line CI-step comment cut to 2 ("the comment is close to 3x the actual config change... reduce to 20%?"); `~1890 MB → ~713 MB` replaced with `~60% smaller` ("rough percentage vs hard mbs"); `shrinking the E2E image transfer to the VPS` had "to the VPS" deleted as noise; `Must be the first step (reinstalls the host daemon)` lost the parenthetical because the step was named "Set up Docker" — the name already said it; a `// Self-calibrate the next run's estimate.` line above a `WriteBaseline(...)` call deleted outright as pure *what*; and a "why metadata can't predict the size" rationale that appeared in two comments collapsed to one. None of these were wrong information — they were unearned length.

**How to apply:** Before committing a comment, test each clause: does it state *why* or just *what*? Is any fact in it already in the adjacent code, the symbol name, the step/method name, or another nearby comment? Could a precise number be a rough one (`~60% smaller` > `1890→713 MB`) without losing the point? If a comment is much longer than the code it sits on, that's a prompt to cut, not a quota to fill. Prefer one trailing-line comment over a multi-line block when one clause suffices. Complements `no-refactor-history-in-code.md` (which bans change-history and defensive framing); this rule is the orthogonal axis of *sizing and redundancy*.
