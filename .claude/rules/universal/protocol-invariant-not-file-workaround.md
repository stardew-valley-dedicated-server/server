# Don't paper over a protocol-level invariant with a file or container-state workaround

When a failure is caused by an invariant enforced by a remote protocol or external service (Steam's "one live login per account", a database's unique constraint, a cluster's leader election, a shared TCP port), do not propose a fix that operates only at the file system, container, or process layer. Copying state into a per-container tmpfs, randomizing volume paths, isolating mounts — none of that helps when the invariant is enforced *above* your code by an authority you don't control. Eliminate the trample at the layer where it's enforced (use disjoint identifiers, fail-fast detection, accept the constraint by design) or accept it.

**Why:** Bug-4 first attempt copied the operator's `session.json` into a per-container tmpfs so each test sidecar got its "own" file. User pushback — "JUST NO, IF SOMETHING TRAMPLES, WE NEED TO ELIMINATE THE TRAMPLE." Right call: SteamKit2's `LogonSessionReplaced` fires when *any* two clients log in with the same Steam account, regardless of which file each one loaded the refresh token from. Steam enforces single-session at the protocol layer; the file is just the carrier of the credential. The correct fix was disjoint accounts (multi-account slicing) plus fail-fast detection of overlap with non-test sidecars on the same host — both operating on *which Steam account* each client uses, not on *where the credential file lives*.

**How to apply:** When proposing a fix for a "two things stepping on each other" failure, identify the *enforcement layer* of the invariant first. Three quick checks: (1) Is the failure produced by code in this repo, or by an external authority (cloud API, kernel, remote daemon, network protocol)? (2) Would the failure still happen if every file path, mount, and container were swapped to fresh ones? (3) Does the invariant key on an *identifier* (username, port, leader-id, primary key) rather than on *storage location*? If any answer points to "external authority enforcing on identifier," then file-level isolation is theatre — you need to either use disjoint identifiers or detect overlap before the second client even tries.
